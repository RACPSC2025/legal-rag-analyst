## 1. Resumen Ejecutivo

El sistema **RAG Analista Legal** es un asistente legal basado en IA generativa diseñado para el análisis crítico de normativa colombiana. Utiliza una arquitectura de **Agente Autónomo** con patrones **CRAG** (Corrective RAG) y **Self-RAG** para garantizar la precisión jurídica y eliminar alucinaciones.

**Componentes principales:**

| Componente | Tecnología | Propósito |
|------------|-----------|-----------|
| **LLM** | AWS Bedrock (Amazon Nova Lite v1, Llama 3.3 70B) | Generación y razonamiento |
| **Embeddings** | Amazon Titan Embed Text v2 (1024 dims) | Representación semántica |
| **Vector Store** | ChromaDB (persistente en SQLite) | Búsqueda semántica |
| **Orquestador** | LangGraph (StateGraph) | Flujo cíclico con reflexión |
| **UI** | Streamlit | Interfaz de usuario dual |
| **PDF Loader** | PyMuPDF + LlamaParse | Extracción de documentos |
| **Reranker** | FlashRank (ms-marco-MiniLM) | Re-ranking post-retrieval |

**Estado general del sistema: Sólido con oportunidades de mejora production-grade.**

### 2.2 Flujo del Grafo CRAG

```
START
  │
  ▼
[retrieve] ────────────→ Recupera docs vía FenixHybridRetriever (BM25 + Vector + FlashRank)
  │
  ▼
[grade_documents] ────→ Evalúa relevancia de cada doc con LLM structured output
  │
  ├── "útil" ──────→ [generate] ──────────→ Two-Step Reading (Rethinking)
  │                     │
  │                     ▼
  │               [check_hallucination] ──→ Verifica respaldo factual
  │                     │
  │                     ├── "limpio" ────→ END ✅
  │                     │
  │                     └── "alucinación" → [generate] (reintento, max 2) → END ⚠️
  │
  └── "no_útil" ───→ [no_answer] ────────→ Respuesta de "no encontrado" → END
```

### 2.3 Estado Tipado (RagState)

```python
class RagState(BaseModel):
    question: str                           # Pregunta del usuario (inmutable)
    documents: List[Document]               # Documentos recuperados (acumulable)
    generation: str                         # Respuesta generada por el LLM
    grade: Literal["útil", "no_útil", "alucinación", "pendiente"]
    hallucination_score: float              # 0.0 (limpio) → 1.0 (máximo riesgo)
    attempts: int                           # Intentos realizados (max 2)
    source_docs: List[str]                  # Fragmentos usados como evidencia
```

**Observaciones:**
- Temperatura fija a `0.0` — correcto para dominio legal
- Falta validación de credenciales al inicio (fail fast)
- Sin lazy initialization de los clientes

### 3.2 Core — LangGraph (`src/core/`)

#### `state.py` — Estado Tipado

**Evaluación:** ✅ Bien diseñado

- Usa `pydantic.BaseModel` con `Field` descriptions
- `Config.arbitrary_types_allowed = True` para `Document`
- Type hints completos con `Literal` para `grade`
- **Bug menor:** No hay validación de que `question` no esté vacío

#### `nodes.py` — Nodos del Grafo

**Evaluación:** ✅ Sólido con hallazgos

| Nodo | Responsabilidad | Técnica |
|------|----------------|---------|
| `retrieve` | Búsqueda híbrida | FenixHybridRetriever (BM25 + Vector + FlashRank) con fallback |
| `grade_documents` | Evalúa relevancia | LLM structured output (`GradeOutput` Pydantic) + fallback si muy estricto |
| `generate` | Genera respuesta | **Two-Step Reading (Rethinking)**: extrae pasajes clave → genera respuesta final |
| `check_hallucination` | Verifica factualidad | LLM structured output (`HallucinationOutput`) — estricto |
| `no_answer` | Respuesta negativa | Mensaje estándar de "no encontrado" |

**Hallazgos por nodo:**

- **`retrieve`:** Fallback a `similarity_search` si falla hybrid — defensivo correcto
- **`grade_documents`:** Fallback a los 3 primeros docs si el grader es muy estricto — buena práctica
- **`generate`:** Two-Step Reading implementa correctamente el patrón Rethinking del Módulo 08 de Rag Mastery
- **`check_hallucination`:** Trunca contexto a 1500 chars — **puede perder evidencia** en chunks largos
- **Bug en `_get_article_metadata()`** (en `pdf_simple.py`): `Path(chunk_text).name` sobre un string de texto crudo → siempre retorna fallback `"document.pdf"`

## 4. Patrones Agenticos Implementados

### 4.1 CRAG (Corrective RAG) — ✅ Implementado

El agente evalúa la calidad de los documentos recuperados antes de generar:

```
retrieve → grade_documents → [útil → generate | no_útil → no_answer]
```

Si los documentos son irrelevantes, el agente no genera — evita contaminación.

### 4.2 Self-RAG (Self-Reflective RAG) — ✅ Implementado

Auto-evaluación post-generación:

```
generate → check_hallucination → [limpio → END | alucinación → generate (retry max 2)]
```

Verifica respaldo factual de la respuesta contra los documentos fuente.

### 4.3 Rethinking (Two-Step Reading) — ✅ Implementado

Del Módulo 08 de Rag Mastery:

1. **Primera lectura:** Extrae pasajes clave y datos técnicos del contexto
2. **Segunda lectura:** Genera respuesta final usando los pasajes identificados + contexto completo

### 4.4 Hybrid Search (BM25 + Vector) — ✅ Implementado

Dense para semántica + Sparse para términos exactos (números de artículo).

### 4.5 FlashRank Reranking — ✅ Implementado

Post-retrieval reranking con modelo cross-encoder local.

### 4.6 Patrones NO implementados (Gaps)

## 9. Veredicto Arquitectónico

### Resumen

El sistema tiene **cimientos sólidos**. Los patrones agenticos fundamentales están bien implementados:

- ✅ **CRAG** con conditional routing y reintentos
- ✅ **Self-RAG** con verificación de alucinaciones post-generación
- ✅ **Two-Step Reading** (Rethinking) para generación precisa
- ✅ **Hybrid Search** (BM25 + Vector + FlashRank)
- ✅ **Pydantic structured output** para validación

Los **gaps principales** son:

1. **Falta de tests** — no hay seguridad contra regression
2. **Query enhancement** — queries vagas pierden precisión
3. **Persistencia de sesiones** — sin checkpointer explícito
4. **Bug de metadata** — `source` siempre fallback
5. **BM25 no escala** — reconstruye en cada init
